Diagnóstico de Base de Dados

Estas opções oferecem um jeito conveniente de definir parâmetros de diagnóstico que podem ajudar você a checar questões de desempenho. Vide o Adaptive Server Enterprise Performance and Tuning Guide para mais informações. Estas opções são específicas de conexão e têm como valor padrão "Desativado."

Para ativar uma opção, clique no botão Ativado dessa opção, e em seguida clique em Aplicar para implementar sua ou suas seleções. Esses valores não são atualizados quando são modificados na base de dados - eles só podem ser escritos.

OBSERVAÇÃO: Os botões Ativado|Desativado para estas opções serão desmarcados depois que você clicar em Aplicar e o comando for mandado para a base de dados, pois o jisql não pode ter certeza do estado atual dessas opções. Por exemplo, você clica em Ativado para "Exibir Estatísticas I/O," clica em Aplicar, e em seguida executa a mesma instrução num lote na Janela Entrada; o jisql não é notificado de que foi mudado o valor da opção.

As mensagens geradas para essas opções são exibidas de acordo com a sua seleção para "Avisos de SQL devem ser apresentados" na guia Diversas da Janela Opções.

OPÇÃO DESCRIÇÃO
Mostrar plano de processamento da consulta (showplan) Quando configurado como "Ativado," gera uma descrição do plano de processamento da consulta que pode ser útil para diagnósticos de desempenho. Esta opção não imprime resultados quando é usada dentro de um procedimento armazenado ou disparador. O padrão é "Desativado."
Compilar mas não executar consultas (noexec)

Quando definido como "Ativado," compila cada consulta, mas não a executa. Uma vez que esta opção esteja definida, nenhum comando subseqüente é executado até que você a desative.
Exibir Estatísticas de Entrada/Saída (statistics io)

Clique em "Ativado" para exibir o número de varreduras, leituras lógicas e físicas para cada tabela referida na instrução de consulta. A saída é sempre exibida na Janela Status.

O padrão é "Desativado."
Exibir informação de cache de subconsulta (statistics subquerycache)
Quando definida como "Desativado," exibe o número de acertos e erros de cache, e o número de linhas no cache de subconsulta para cada subconsulta. O padrão é "Desativado."
Exibir tempos de análise (parse), compilação e execução (statistics time)
 Quando definido como "Ativado", exibe o tempo levado para analisar e compilar cada comando; para cada passo do comando,é exibido o tempo levado para executá-lo. Os tempos são dados em milissegundos e em tiques de tempo, cujo valor exato depende de cada computador. O padrão é "Desativado."
