Menu Arquivo

OPÇÃO
DESCRIÇÃO
Ler Arquivo de lote

Permite-lhe executar arquivos de lote preparados. Dependendo do estado definido na guia Opções de Janela de Entrada, um arquivo pode ser lido na janela de entrada, onde ele podee ser editado e em seguida executado através do botão Ir ou da opção Executar no menu Consulta. O Delimitador de Linha de Comando pode ser configurado na guia Opção de Janela de Entrada.

Para executar o lote silenciosamente em segundo plano, basta marcar a opção apropriada na guia Opção de Janela de Entrada.
Salvar Janela de Entrada em Arquivo Salva o conteúdo atual da Janela de Entrada num arquivo. Mais tarde, você pode lê-lo e executá-lo de novo.
Salvar o Buffer de Saída em Arquivo

Cada ResultSet é armazenado num buffer de Saída. Esta opção permite-lhe salvar o conteúdo do buffer de Saída num arquivo. Esta é uma abordagem simples para se salvar dados armazenados na base de dados. Se os dados forem apresentados como um Componente de Tabela, a guia Opção de Janela de Saída lhe dá a opção de definir o campo do delimitador. Isso lhe permite preparar o arquivo de saída para usá-lo mais tarde com bcp para ler os dados de volta numa tabela de base de dados. O delimitador de linha é sempre LineFeed.

Se os dados forem apresentados como Texto, são armazenados tais como você os vê na Janela de Saída.
Salvar o Buffer de Histórico em Arquivo Cada comando executado por você é armazenado num buffer de Histórico que é exibido quando você clica no ícone Histórico. Você pode editar o seu conteúdo (por exemplo, acrescentar comentários) e salvar o buffer de Histórico num arquivo para fins de logon ou para ajudá-lo a solucionar falhas.
Imprimir ResultSets em Estilo ASCII

Envia o atual ResultSet a uma impressora. Os dados são apresentados como texto ASCII.

OBSERVAÇÃO:  Se você modificar os dados na Janela de Saída, (por exemplo, redimensionando ou reordenando as colunas) essas mudanças não se refletirão na saída impressa.
Imprimir ResultSets em Estilo JTable
(apenas jdk 1.2)

Envia o atual ResultSet a uma impressora. Os dados são apresentados como uma JTable processada.

OBSERVAÇÃO:  Se você modificar os dados na Janela de Saída, (por exemplo, redimensionando ou reordenando as colunas) essas mudanças não se refletirão na saída impressa.
Sair Fecha todas as janelas de sessão (o que também fecha as conexões associadas com base de dados) e encerra o processo de aplicação.
