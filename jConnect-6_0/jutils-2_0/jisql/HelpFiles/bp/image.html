Colunas de Imagens

Se você selecionar dados que contêm uma coluna de imagens e o ResultSet for apresentado numa tabela e não como texto (Saída em formato tabela ou em formato texto), a coluna de imagens não mostrará os dados binários. Em vez disso, a tabela será denominada "Clique em mim - Sou uma imagem."

Clicar na célula abre uma nova janela com quatro opções:

  • Mostrar dados de imagem como uma figura - Se os dados de imagem representarem uma figura em formato gif ou jpg, a imagem será exibida numa janela de imagem.
  • Mostrar dados de imagem como dados hexadecimais - Isto abre uma janela que mostra os dados em formato hexadecimal; esta é a maneira como seriam exibidos no isql.
  • Salvar dados de imagem num Arquivo - Os dados serão salvos num arquivo para uso posterior. Isto é útil, por exemplo, no caso de um arquivo de som ou de um fluxo de vídeo que você queira visualizar com algum outro programa.
  • Não fazer nada - Fecha novamente a janela de opções.

Colunas de Objetos

Se um ResultSet contiver uma ou mais colunas com objetos Java, aparece um texto na tabela do ResultSet dizendo '"Clique em mim- Sou um Objeto." Quando você faz isso, o jisql executa o método toString() definido nesse objeto e apresenta os valores que o método retorna como dados na tabela. Se os objetos usarem um método que não seja o toString() para imprimir seus dados, digite esse nome na guia Opções de Janela de Saída da Janela Opção; ou seja, mude o "Método usado no objeto DB para exibir dados" para o nome do método usado.

Se a opção "Extrair automaticamente os dados do Objeto Java DB" estiver ativada, o jisql extrai os dados imediatamente quando detetar objetos Java no ResultSet. Você não vê a mensagem "Clique em mim" e não tem de clicar para receber os dados. Isso desacelera a criação da tabela ResultSet, pois o jisql tem de fazer uma viagem de ida e volta até a base de dados para cada linha, a fim de executar o método toString() (ou equivalente) com o objeto.
