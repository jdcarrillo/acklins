Opções de Base de Dados

Estas opções oferecem um jeito conveniente de definir parâmetros usados com freqüência. Vide os manuais Transact-SQL para detalhes acerca do uso e das funções dessas definições. Estes valores não são atualizados quando são modificados na base de dados.

OBSERVAÇÃO: Os botões Ativado|Desativado para opções serão desmarcados depois que você clicar em Aplicar e o comando for enviado à base de dados, pois o jisql não pode ter certeza sobre o estado atual dessas opções. Por exemplo, você clica em Ativado para "Use Transações Encadeadas," clica em Aplicar, e então executa a mesma instrução na Janela de Entrada; o jisql não é notificado de que mudou o valor da opção.

Clique em Aplicar para salvar as mudanças.

OPÇÃO DESCRIÇÃO
Número Máximo de Linhas a retornar (rowcount) Faz a consulta (selecionar, inserir, atualizar ou excluir) parar o processamento depois que o número especificado de linhas é atingido. Para desativar esta opção, digite 0 (zero).
Limite de tamanho de Texto ou Imagem a retornar (textsize)

Especifica o limite, em bytes, dos dados de texto ou imagem a serem retornados com uma instrução selecionar.
Definir o nível de isolamento da transação (0 - 3)

Especifica até que ponto os dados podem ser acessados por outros usuários durante uma transação.

0 - Outras transações são impedidas de modificar os dados até que a transação seja confirmada. Porém, outras transações ainda podem ler os dados não confirmados.

1 - Previne que uma transação modifique uma linha enquanto uma segunda transação lê a linha antes que a primeira transação confirme a mudança.

2 - Previne que uma transação leia uma linha enquanto uma segunda transação modifica a mesma linha.

3 - Previne que uma transação leia um conjunto de linhas que respondam a uma consulta, enquanto uma segunda transação modifica os dados. Isto garante que os dados lidos por uma transação sejam válidos até o fim dessa transação.
Mostrar número de linhas afetadas Ativa ou desativa a exibição das linhas afetadas por uma instrução inserir, atualizar ou excluir. O padrão é "Desativado."
Usar Transações Encadeadas (chained) Em modo encadeado, todos os comandos de recuperação e modificação de dados(excluir, inserir, abrir, extrair, selecionar e atualizar) iniciam uma transação. Este comportamento é incompatível com muitos aplicativos T-SQL. O padrão é "Desativado."
Reconhecer identificadores delimitados (quoted_identifier) Esta opção permite que a base de dados reconheça identificadores delimitados. Cada vez que você usa o identificador citado numa instrução, você deve colocá-lo entre aspas duplas. Quando esta opção está configurada como"Ativado," não use aspas duplas para delimitar seqüências de caracteres ou datas; use em vez disso aspas simples. Para inserir aspas simples numa coluna, use duas aspas simples consecutivas. O padrão é "Desativado."

Para mais informações, vide oTransact-SQL User's Guide.
