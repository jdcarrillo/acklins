数据库诊断

用这些选项可以简便地设置诊断参数。这些参数可帮助您检查性能问题。详细信息参见 Adaptive Server Enterprise 性能和调优指南。这些选项因连接而异，缺省值为“关闭”。

要激活某个选项，请单击该选项的打开按钮，然后单击应用实施选择的选项。当在数据库中这些值发生变化时，并不更新这些值 - 它们是只写的。

注释： 单击应用并将命令发送到数据库后，这些选项的打开|关闭按钮无标记，这是因为 jisql 不能确定这些选项的当前状态。例如，单击“显示 I/O 统计信息”的打开按钮并单击应用，然后在“输入窗口”中执行批处理中的同一语句后，并没有通知 jisql 选项的值已改变。

选择了选项窗口的杂类选项卡上“应该显示 SQL 警告”，就将显示为这些选项生成的消息。

选项 说明
显示查询的处理计划 (showplan) 设置为“打开”时，将生成查询处理计划的说明。该说明在性能诊断时有用。在存储过程或触发器内部使用该选项时，该选项不输出结果。缺省为“关闭”。
编译但不执行查询 (noexec)

设置为“打开”时，将编译每个查询但不执行查询。一旦设置了此选项，则只有在“关闭”该选项后才执行后续命令。
显示 I/O 统计信息 (statistics io)

单击“打开”，显示查询语句中引用的每个表的扫描数、逻辑读取数和物理读取数。输出始终显示在“状态窗口”中。

缺省为“关闭”。
显示子查询高速缓存信息 (statistics subquerycache)
 设置为“打开”时，将显示每个子查询的高速缓存命中次数、高速缓存遗失次数以及子查询高速缓存中的行数。缺省为“关闭”。
显示语法分析时间、编译时间和执行时间 (statistics time)
 设置为“打开”时，将显示语法分析和编译每个命令所用的时间；将显示执行命令的每个步骤所用的时间。时间按毫秒和时间周期给出，其精确度取决于计算机。缺省为“关闭”。
