Diagnostics de base de données

Ces options permettent de définir facilement des paramètres pour contrôler les éventuels problèmes de performance. Pour plus d'informations, reportez-vous au document Adaptive Server Enterprise - Performances et optimisation. Ces options sont spécifiques de la connexion et sont désactivées par défaut ("Off").

Pour activer une option, cliquez sur le bouton On qui lui est associé puis cliquez sur Appliquer pour rendre vos sélections effectives. Lorsque ces valeurs changent dans la base de données, elles ne sont pas mises à jour, elles sont en écriture seulement.

REMARQUE Les boutons On|Off de ces options sont désélectionnés une fois que vous avez cliqué sur Appliquer et la commande est envoyée à la base de données, car jisql ne peut pas être sûr de l'état en cours de ces options. Par exemple, si vous cliquez sur On pour "Afficher les statistiques d'E/S", puis sur Appliquer et que vous exécutez la même instruction dans un batch dans la fenêtre de saisie, jisql n'est pas averti que la valeur de l'option a changé.

Les messages générés pour ces options s'affichent selon votre sélection pour l'option "Des avertissements SQL doivent être affichés" dans l'onglet Options diverses de la fenêtre des options.

OPTION DESCRIPTION
Afficher le plan de traitement des requêtes (showplan) Définie à "On" (activée), cette option génère une description du plan de traitement pour la requête, utilisable lors du diagnostic des performances. Cette option n'imprime aucun résultat lorsqu'elle est utilisée dans le cadre d'une procédure stockée ou d'un trigger. La valeur par défaut est "Off".
Compiler les requêtes mais ne pas les exécuter (noexec)

Définie à "On," cette option compile chaque requête, sans l'exécuter. Une fois que cette option est définie, aucune commande suivante n'est exécutée tant que vous ne la désactivez pas.
Afficher les statistiques d'E/S (statistics io)

Cliquez sur "On" pour afficher le nombre d'analyses, de lectures logiques et de lectures physiques pour chaque table référencée dans l'instruction de requête. Les résultats s'affichent toujours dans la fenêtre d'état.

La valeur par défaut est "Off".
Afficher les informations de cache de sous-requête (statistics subquerycache)
Définie à "On," cette option affiche le nombre de présences dans le cache, les non-présences, ainsi que le nombre de lignes dans le cache de sous-requête pour chaque sous-requête. La valeur par défaut est "Off".
Afficher les temps d'analyse, de compilation et d'exécution (statistics time)
 Définie à "On," cette option affiche le temps consacré à l'analyse et à la compilation de chaque commande ; pour chaque phase de la commande, elle affiche le temps passé à l'exécution. Le temps est indiqué en millisecondes et en impulsions, la valeur exacte dépendant de la machine. La valeur par défaut est "Off".
