Options de la base de données

Ces options permettent de définir facilement les options les plus utilisées de la base de données. Pour plus d'informations sur l'utilisation de ces paramètres et leurs fonctions, reportez-vous à la documentation Transact-SQL. Ces valeurs ne sont pas mises à jour lorsqu'elles changent dans la base de données.

REMARQUE Les boutons On|Off des options sont désélectionnés une fois que vous avez cliqué sur Appliquer et la commande est envoyée à la base de données, car jisql ne peut pas être sûr de l'état en cours de ces options. Par exemple, si vous cliquez sur On pour "Utiliser les transactions chaînées", puis sur Appliquer et que vous exécutez la même instruction dans la fenêtre de saisie, jisql n'est pas averti que la valeur de l'option a changé.

Cliquez sur Appliquer pour enregistrer vos modifications.

OPTION DESCRIPTION
Nombre maximal de lignes à renvoyer (rowcount) Oblige la requête (select, insert, update ou delete) à arrêter le traitement une fois que le nombre de lignes spécifié est atteint. Pour désactiver cette option, entrez 0 (zéro).
Taille maximale texte|image à renvoyer (textsize)

Spécifie la limite, en octets, des données text ou image à renvoyer avec une instruction select.
Définir le niveau d'isolement des transactions (0 - 3)

Indique le niveau de données auquel les utilisateurs peuvent accéder lors d'une transaction.

0 - Les autres transactions sont bloquées et ne peuvent pas modifier les données tant que la transaction n'est pas validée. Toutefois, elles peuvent toujours lire les données non validées.

1 - Empêche une transaction de modifier une ligne pendant qu'une seconde transaction lit la ligne, et ce tant que la première transaction n'a pas validé les modifications.

2 - Empêche une transaction de lire une ligne pendant qu'une seconde transaction modifie cette même ligne.

3 - Empêche une transaction de lire un ensemble de lignes qui répond à une requête, pendant qu'une seconde transaction modifie les données. Cela garantit que les données lues par une transaction sont valides jusqu'à la fin de cette transaction.
Afficher le nombre de lignes concernées Active ou désactive l'affichage des lignes sur lesquelles s'appliquent une instruction insert, update ou delete. Par défaut, cette option est désactivée ("Off").
Utiliser les transactions chaînées (chained) En mode chaîné, toutes les commandes de recherche et de modification de données (delete, insert, open, fetch, select et update) se trouvent au début d'une transaction. Ce comportement est incompatible avec de nombreuses applications T-SQL. Par défaut, cette option est désactivée ("Off").
Reconnaître les identificateurs délimités (quoted_identifier) Cette option permet à la base de données de reconnaître les identificateurs délimités. Chaque fois que vous utilisez un identificateur délimité dans une instruction, vous devez le placer entre guillemets. Lorsque cette option est activée ("On"), ne placez pas de caractères ni de chaîne de date entre guillemets ; utilisez à la place des apostrophes. Pour insérer une apostrophe dans une colonne, utilisez deux apostrophes consécutives. Par défaut, cette option est désactivée ("Off").

Pour plus d'informations, reportez-vous au document Guide de l'utilisateur Transact-SQL.
