Options de la fenêtre de saisie

  • Délimiteur de commande des fichiers batch
    Le délimiteur de commande permet de signaler la fin d'une instruction et de transmettre cette dernière au serveur de base de données afin qu'elle y soit exécutée. Le délimiteur par défaut est 'go', généralement utilisé avec Transact-SQL.

    Pour utiliser un délimiteur de commande différent, entrez un autre mot-clé.

    REMARQUE :    Tous les blancs étant supprimés, les délimiteurs ' go ' et 'go' sont traités sans distinction.

  • Afficher batch dans la fenêtre de saisie ?
    Si cette case est cochée (option par défaut), le fichier batch importé s'affiche dans la fenêtre de saisie où vous pouvez l'éditer et l'exécuter en cliquant sur le bouton Soumettre ou en sélectionnant l'option Exécuter du menu requête. Sinon, le fichier batch est exécuté en arrière-plan. Dans les deux cas, les messages de résultats et/ou d'état sont affichés.

  • Annuler batch à la première exception ?

    Si une exception SQL se produit en cours d'exécution d'un fichier batch volumineux alors que cette case est cochée (option par défaut), jisql annule le batch.
