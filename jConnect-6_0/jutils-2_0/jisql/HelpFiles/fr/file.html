Menu Fichier

OPTION
DESCRIPTION
Lire fichier batch

Cette option permet d'exécuter des fichiers batch préparés. Selon l'état défini dans l'onglet Options de la fenêtre de saisie, il est possible de lire un fichier dans la fenêtre de saisie, de le modifier, puis de l'exécuter en cliquant sur le bouton Soumettre ou en sélectionnant l'option Exécuter du menu Requête. Le délimiteur de ligne de commande peut être défini dans l'onglet Options de la fenêtre de saisie.

Pour exécuter le fichier batch en arrière-plan, en mode non détaillé, il suffit de cocher l'option correspondante dans l'onglet Options de la fenêtre de saisie.
Enregistrer la fenêtre de saisie dans un fichier Cette option enregistre le contenu courant de la fenêtre de saisie dans un fichier. Vous pourrez ensuite le lire et exécuter les données qu'il contient.
Enregistrer le buffer de résultats dans un fichier

Chaque jeu de résultats est stocké dans un buffer de résultats. Cette option permet donc d'enregistrer le contenu de ce buffer dans un fichier. Il est ainsi facile d'enregistrer les données stockées dans la base. Si ces données sont présentées sous forme de Table, utilisez l'onglet Options de la fenêtre de résultats pour définir le délimiteur de champ. Vous pouvez ainsi préparer le fichier de résultats avec la commande bcp afin de relire ultérieurement les données dans une table de base de données. Le délimiteur est toujours LineFeed.

Si les données sont affichées sous forme de Texte, elles sont stockées telles qu'elles apparaissent dans la fenêtre de résultats.
Enregistrer le buffer d'historique dans un fichier Chaque commande exécutée est enregistrée dans le buffer de l'historique qui s'affiche lorsque vous cliquez sur le bouton Historique. Vous pouvez modifier le contenu de ce buffer (par exemple, ajouter des commentaires) et l'enregistrer dans un fichier journal qui aidera le Support Technique à reproduire des pannes, le cas échéant.
Imprimer le style ASCII des jeux de résultats

Cette option envoie le jeu de résultats courant à une imprimante. Les données sont alors présentées sous forme de texte ASCII.

REMARQUE : Si vous modifiez les données dans la fenêtre de saisie (par exemple en redimensionnant ou en réorganisant les colonnes), ces modifications n'apparaissent pas sur le document imprimé.
Imprimer les jeux de résultats dans le style JTable
(jdk 1.2 uniquement)

Cette option envoie le jeu de résultats courant à une imprimante. Les données sont présentées sous forme de JTable avec rendu.

REMARQUE : Si vous modifiez les données dans la fenêtre de saisie (par exemple en redimensionnant ou en réorganisant les colonnes), ces modifications n'apparaissent pas dans le document imprimé.
Quitter Cette option ferme toutes les fenêtres de session (ce qui met fin également aux connexions à la base de données) ainsi que l'application.
