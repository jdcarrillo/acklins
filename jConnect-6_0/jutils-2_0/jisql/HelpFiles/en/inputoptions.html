Input Window Options

  • Command delimiter to be used in Batches
    The command delimiter marks the end of a statement and sends it to the database server for execution.The command delimiter defaults to 'go', which is the delimiter for Sybase's Transact-SQL.

    To use a command delimiter different than 'go,' enter a different keyword.

    NOTE:    Blanks will be stripped, which means ' go ' and 'go' are handled identically.

  • Show a batch read from a file in the Input Window?
    When using the default (marked), an imported batch file displays in the Input Window ready for you to edit and execute by clicking the Go button or the Query Menu's Execute option. If it is not marked, the batch will be executed in the background. The output and/or status messages will display in either case.
  • Cancel Batch on the first Exception?

    When this option is checked (default), and a SQL Exception occurs while a large batch is running, jisql cancels the batch.
