Datenbankoptionen

Hier können Sie häufig benutzte Parameter einstellen. Im Benutzerhandbuch für Transact-SQL finden Sie weitere Hinweise zum Einsatz und zu den Funktionen dieser Einstellungen. Diese Werte werden nicht aktualisiert, wenn sie sich in der Datenbank ändern.

HINWEIS: Die Schaltflächen Ein|Aus für die Parameter werden ent-markiert, wenn Sie auf Übernehmen klicken, und der Befehl wird an die Datenbank geschickt, weil jisql nicht sicher sein kann, welchen Status diese Parameter gerade haben. Beispiel: Sie klicken auf Ein für "Verkettete Transaktion", dann auf Übernehmen, führen dann dieselbe Anweisung im Eingabefenster aus. In diesem Fall wird jisql nicht verständigt, dass der Wert der Option geändert wurde.

Klicken Sie auf Übernehmen, um die Änderungen zu speichern.

OPTION BESCHREIBUNG
Maximale Anzahl an Zeilen (Zeilenanzahl) Bewirkt, dass die Abfrage (select, insert, update oder delete) die Verarbeitung stoppt, nachdem die angegebene Anzahl von Zeilen betroffen ist. Um diese Option auszuschalten, geben Sie 0 (Null) ein.
Begrenzung der text|image-Größe (Textgröße)

Gibt in Byte den Grenzwert der text- oder image-Daten an, die mit einer select-Anweisung zurückgegeben werden sollen.
Transaktions-Isolationsstufe setzen (0 - 3)

Gibt den Grad an, mit dem während einer Transaktion auf Daten zugegriffen werden kann.

0 - Andere Transaktionen können die Daten erst ändern, wenn die laufende Transaktion festgeschrieben wird. Nicht festgeschriebene Daten können von anderen Transaktionen aber gelesen werden.

1 - Eine Transaktion kann eine Zeile nicht ändern, während eine zweite Transaktion die Zeile liest, solange die erste Transaktion die Änderung nicht festgeschrieben hat.

2 - Eine Transaktion kann eine Zeile nicht lesen, während eine zweite Transaktion dieselbe Zeile ändert.

3 - Eine Transaktion kann eine Zeilengruppe nicht lesen, die von einer Abfrage verlangt wird, während eine zweite Transaktion die Daten ändert. Damit wird gewährleistet, dass Daten, die von einer Transaktion gelesen werden, bis zum Ende dieser Transaktion gültig sind.
Anzeige 'number of rows affected' Damit wird die Anzeige der Anzahl von Zeilen ein- und ausgeschaltet, die durch eine insert-, update- oder delete-Anweisung betroffen sind. Der Standard ist "Aus".
Benutze verkettete Transaktionen (chained) Im Modus "chained" beginnen alle Datenabruf- und Datenänderungsbefehle (delete, insert, open, fetch, select und update) eine Transaktion. Dieses Verhalten ist mit vielen T-SQL-Anwendungen nicht kompatibel. Der Standard ist "Aus".
Begrenzte Bezeichner erkennen (quoted_identifier) Mit dieser Option kann die Datenbank begrenzte Bezeichner erkennen. Jedesmal wenn Sie angeführte Bezeichner in einer Anweisung verwenden, müssen Sie sie in Anführungszeichen setzen. Wenn diese Option auf "On" gesetzt ist, dürfen Sie Zeichenfolgen oder Datumsfolgen nicht in Anführungszeichen setzen, sondern müssen Apostrophe verwenden. Um einen Apostroph in eine Spalte zu setzen, benutzen Sie zwei aufeinanderfolgende Apostrophe. Der Standard ist "Aus".

Weitere Hinweise finden Sie in der Dokumentation SQLTransact-SQL Benutzerhandbuch.
