Datenbankdiagnose

Diese Optionen bieten eine bequeme Methode, um Diagnoseparameter zu setzen, mit denen Sie Probleme der Performance lösen können. Weitere Hinweise finden Sie in der Dokumentation Performance und Tuning für Adaptive Server Enterprise. Diese Optionen sind verbindungsspezifisch und haben den Standardwert "Aus".

Um eine Option zu aktivieren, klicken Sie auf die Schaltfläche Ein der Option und dann auf Übernehmen, um Ihre Auswahl zu aktivieren. Diese Werte werden nicht aktualisiert, wenn sie in der Datenbank geändert werden - sie sind wirte-only.

HINWEIS: Die Schaltflächen Ein|Aus für diese Optionen werden entmarkiert, nachdem Sie auf Übernehmen klicken, und der Befehl wird an die Datenbank geschickt, weil jisql nicht sicher sein kann, welchen Status diese Parameter gerade haben. Beispiel: Sie klicken auf Ein für "I/O-Statistik anzeigen", dann auf "Übernehmen". Danach führen Sie dieselbe Anweisung in einer Anweisungsfolge im Eingabefenster aus. jisql wird nicht informiert, dass der Wert der Option geändert wurde.

Die für diese Optionen generierten Nachrichten werden gemäß Ihrer Auswahl für die Option "SQL-Warnungen anzeigen als" im Register Sonstiges im Fenster Optionen ausgegeben.

OPTION BESCHREIBUNG
Abfrageplan anzeigen (showplan) Wenn die Option auf "Ein" gesetzt ist, wird eine Beschreibung des Verarbeitungsplans für die Abfrage erstellt, die bei der Performancediagnose nützlich sein kann. Diese Option gibt keine Ergebnisse aus, wenn sie innerhalb einer gespeicherten Prozedur oder eines Triggers verwendet wird. Der Standard ist "Aus".
Kompilieren, aber Abfragen nicht ausführen (noexec)

Wenn die Option auf "Ein" gesetzt ist, kompiliert jede Abfrage, wird aber nicht ausgeführt. Wenn diese Option gesetzt wurde, werden keine darauf folgenden Befehle ausgeführt, bis Sie sie wieder auf "Aus" schalten.

I/O-Statistik anzeigen (statistics io)

Klicken Sie auf "Ein", um die Anzahl von Scans, logischen Lesevorgängen und jede Tabelle anzuzeigen, die in der Abfrageanweisung referenziert wird. Die Ausgabe erfolgt immer im Statusfenster.
Der Standard ist "Aus".
Unterabfrage-Cacheinformation anzeigen (statistics subquerycache)
 Wenn diese Option auf "Ein" gesetzt ist, wird die Anzahl der Cachetreffer, der Cachefehlschläge, sowie die Anzahl der Zeilen im Unterabfragencache für jede Unterabfrage angezeigt. Der Standard ist "Aus".
Anzeige von Analyse-, Kompilierungs- und Ausführungszeiten (statistics time)
 Wenn diese Option auf "Ein" gesetzt ist, wird die Zeit angezeigt, die erforderlich war, um jeden Befehl syntaktisch zu analysieren und zu kompilieren. Für jeden Schritt des Befehls wird die Zeit angezeigt, die für seine Ausführung erforderlich war. Die Zeitangaben erfolgen in Millisekunden und in Zeittakten, deren exakter Wert vom Rechner abhängt. Der Standard ist "Aus".
