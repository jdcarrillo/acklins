Fenster

jisql kann gleichzeitig null, eine oder mehrere Verbindungen offen halten, die in einem eigenen Sitzungsfenster angezeigt werden. Jedes neue Sitzungsfenster hat folgende Elemente:

  • Ein Eingabefenster, in dem Sie Ihre Anweisungen eingeben
  • Ein Ausgabefenster, das die Ergebnisse von ausgeführten Anweisungen anzeigt
  • Ein Statusfenster unter dem Ausgabefenster, das Fehler bei der Abfragenverarbeitung und Informationsmeldungen anzeigt
  • Die Symbolleiste mit dem Symbol Go zum Ausführen von Anweisungen, einer Auswahlliste der Datenbanken, die sich auf dem Server befinden, mit dem die Verbindung aufrecht ist, sowie das Symbol für das Ablauffenster, mit dem Sie vorher ausgeführte Abfragen wieder aufrufen können.
