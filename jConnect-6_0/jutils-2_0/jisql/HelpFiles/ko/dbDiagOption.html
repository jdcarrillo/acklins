데이터베이스 진단

이러한 옵션을 사용하여 성능 문제를 확인하는 데 사용되는 진단 매개변수를 쉽게 설정할 수 있습니다. 자세한 내용은 Adaptive Server Enterprise 성능과 조정 안내서를 참조하십시오. 이 옵션은 연결별로 적용되며 기본값은 "해제"입니다.

옵션을 설정하려면 해당 옵션의 설정 단추를 선택한 다음 적용을 누릅니다. 이 값은 데이터베이스에서 변경해도 업데이트되지 않습니다. 이 값은 쓰기 전용입니다.

참고: 적용을 눌러 명령이 데이터베이스로 전달된 후에는 jisql이 옵션의 현재 상태를 확인할 수 없기 때문에 옵션의 설정|해제 단추가 선택되지 않은 상태로 나타납니다. 예를 들어 "입/출력 통계 표시"를 설정으로 선택하고 적용을 누른 후 입력 창의 배치에서 동일한 구문을 실행하면, jisql이 옵션의 값이 변경되었다는 것을 인식하지 못합니다.

이러한 옵션에 대해 생성된 메시지를 표시하려면 옵션 창의 기타 탭에서 "SQL 경고 메시지 표시" 옵션을 선택합니다.

옵션
설명
질의 처리 계획 표시(showplan)
이 옵션을 설정하면 질의 처리 계획에 대한 설명을 생성합니다. 질의 처리 계획 설명은 성능 진단에 유용합니다. 이 옵션이 저장 프로시저나 트리거 내부에서 사용되면 결과 메시지가 출력되지 않습니다. 기본값은 "해제"입니다.
질의를 컴파일만 하고 실행하지는 않음(noexec)

이 옵션을 설정하면 각 질의를 컴파일만 하고 실행하지는 않습니다. 이 옵션을 설정하면 "해제"로 설정할 때까지 후속 명령이 실행되지 않습니다.
입/출력 통계 표시(statistics io)

이 옵션을 설정하면 질의 문에 참조된 각 테이블에 대한 스캔, 논리적 읽기, 물리적 읽기 수가 표시됩니다. 결과는 항상 상태 창에 표시됩니다.

기본값은 "해제"입니다.
하위 질의 캐시 정보 표시(statistics subquerycache)
이 옵션을 설정하면 캐시 적중 수, 실패 수, 각 하위 질의에 대한 캐시의 행 수를 표시합니다. 기본값은 "해제"입니다.
구문 분석, 컴파일, 실행 시간 표시(statistics time)
이 옵션을 설정하면 각 명령의 구문 분석 및 컴파일에 걸린 시간을 표시하며, 명령의 각 단계에 대한 실행 시간을 표시합니다. 시간은 100분의 1초 단위로 표시되며 정확한 시간은 컴퓨터의 성능에 따라 다릅니다. 기본값은 "해제"입니다.
