이미지 열

이미지 열을 포함하는 데이터를 선택하여 결과 집합이 텍스트 형식(표 형식과 텍스트 형식의 출력)이 아닌 표 형식으로 나타날 경우에는 이미지 열에 이진 데이터가 표시되지 않습니다. 대신 테이블에 "이미지를 누르십시오."라는 레이블이 표시됩니다.

셀을 누르면 네 개의 선택 사항을 제공하는 새로운 창이 열립니다.

  • 그림으로 이미지 보기 - 이미지 데이터가 gif나 jpg 형식의 그림일 경우 이미지 창에 나타납니다.
  • 16진수 데이터로 이미지 보기 - 16진수 형식으로 데이터를 표시하는 창을 엽니다. 이것은 isql에서 표시하는 방식입니다.
  • 파일에 이미지 저장 - 나중에 사용할 수 있도록 데이터를 파일에 저장합니다. 예를 들어, 사운드 파일이나 비디오 스트림을 다른 프로그램에서 열고자 할 때 유용합니다.
  • 아무 것도 실행하지 않음 - 선택 창을 닫습니다.

객체 열

결과 집합에 Java 객체를 포함한 열이 있으면 결과 집합의 테이블에 "객체를 누르십시오."라는 텍스트가 나타납니다. 객체를 누르면 jisql이 해당 객체에 정의된 toString() 메소드를 실행하여 메소드가 테이블 데이터로 반환한 값을 나타냅니다. 객체가 toString() 외의 다른 메소드를 사용하여 데이터를 출력할 경우 옵션 창의 출력 창 옵션 탭에 해당 이름을 입력하십시오. 즉, "데이터를 나타내기 위해 DB 객체에 사용된 메소드"를 해당 메소드 이름으로 변경합니다.

"DB Java 객체에서 자동으로 데이터 추출" 옵션을 설정하면 jisql은 결과 집합에서 Java 객체를 검출하는 즉시 데이터를 추출합니다. "객체를 누르십시오."라는 메시지가 표시되지 않으면 데이터 수신을 위해 객체를 누를 필요가 없습니다. 객체를 누르면 객체의 toString() 메소드나 이에 상응하는 메소드를 실행하기 위해 jisql이 데이터베이스의 모든 행을 왕복해야 하므로 결과 집합 테이블 작성 속도가 느려집니다.
